## 1. 项目概述
### 1.1 项目目标
基于原建设方案，明确可量化的落地目标：
- 实现对特朗普政府决策相关的5大类27个核心指标的实时采集（数据延迟≤5分钟）
- 多因子影响权重模型的历史事件回溯准确率≥85%
- 预警系统误报率≤15%，预警响应时间≤1分钟
- 支持至少1000个并发用户的仪表盘访问
- 提供可一键部署的容器化环境，部署周期≤2小时

### 1.2 项目范围
**包含模块**：
1.  多源数据采集与清洗流水线
2.  分布式数据存储集群
3.  实时分析引擎与权重评估模型
4.  交互式可视化仪表盘与复盘工作台
5.  智能预警与多渠道推送系统

**排除范围**：
- 第三方数据源付费授权（仅提供对接规范）
- 硬件基础设施采购
- 企业级身份认证系统（仅提供OAuth2对接接口）

#### 3.1.1 公开API对接示例（Truth Social）
```python
import requests
import time

def fetch_truth_social_posts(count=100):
    url = "https://api.truthsocial.com/v1/timelines/user/realDonaldTrump"
    headers = {
        "Authorization": "Bearer YOUR_API_KEY",

    try:
        response = requests.get(url, headers=headers, timeout=10)
        response.raise_for_status()
        return response.json()
    except requests.exceptions.RequestException as e:
        print(f"API请求失败: {e}")
        return None
```

#### 3.1.2 网页爬虫配置
针对无API的数据源，采用Scrapy集群实现，配置防反爬策略：
- 随机请求头
- 随机请求间隔（1-3秒）
- 代理池轮换
- robots.txt合规性检查

### 3.2 数据清洗流水线
基于Great Expectations + Polars实现标准化清洗：
1.  格式校验：验证数据类型、必填字段完整性
2.  异常值处理：基于3σ原则剔除极端异常值
3.  缺失值填充：采用线性插值或同周期均值填充
4.  多源数据合并：对同一指标采用加权平均融合多源数据

### 3.3 采集调度策略
| 数据类型 | 采集频率 | 调度工具 |
|----------|----------|----------|
| 实时数据（价格、言论） | 1分钟/次 | Apache Flink |
| 每日数据（民调、产量） | 每日凌晨2点 | Airflow |
| 每周数据（月报、周报） | 每周一凌晨1点 | Airflow |

## 4. 数据分析引擎详细设计
### 4.1 多因子权重评估模型
#### 4.1.1 数学模型实现
```python
import numpy as np

def calculate_composite_index(factor_scores, dynamic_weights):
    """
    计算综合影响指数
    :param factor_scores: 各因子得分字典
    :param dynamic_weights: 动态权重字典
    :return: 综合指数(0-100)
    """
    composite = 0
    for factor, score in factor_scores.items():
        composite += score * dynamic_weights.get(factor, 1.0)
    return np.clip(composite / len(factor_scores) * 100, 0, 100)

# 动态权重示例（选举前3个月）
election_period_weights = {
    "geopolitical": 0.2,
    "domestic_political": 0.4,
    "financial_market": 0.2,
    "energy_market": 0.15,
    "decision_team": 0.05
}
```

#### 4.1.2 模型训练流程
1.  基于2017-2025年特朗普政府历史决策事件构建训练集
2.  采用线性回归模型训练各因子的基础权重
3.  基于当前事件上下文动态调整权重（如选举周期、油价高位等）
4.  模型准确率每季度重新训练更新

### 4.2 模式识别引擎
基于Scikit-learn实现分类模型，识别4种决策触发模式：
| 模式 | 模型选型 | 特征工程 |
|------|----------|----------|
| 选举危机驱动型 | 逻辑回归 | 支持率、选举倒计时、政策倾向 |
| 市场崩盘驱动型 | 孤立森林 | 股市跌幅、债市收益率、VIX指数 |
| 地缘压力过载型 | 关联规则挖掘 | 油价、海峡通行量、军事事件 |
| 个人风格放大型 | 时序聚类 | 官员变动、言论倾向、决策频率 |

### 4.3 复盘分析工作台
#### 4.3.1 核心功能
1.  **时间轴可视化**：基于D3.js实现交互式时间轴，支持按因子维度筛选
2.  **因子贡献度分析**：雷达图展示各因子对当前事件的影响占比
3.  **反事实模拟器**：允许用户调整单个因子数值，模拟不同决策结果
4.  **报告生成**：一键导出Markdown/PDF格式的分析报告
